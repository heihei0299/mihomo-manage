mihomo-manager 术语表

本文档是 mihomo-manager 的规范术语。所有课程和参考文档均使用此处的定义。

核心概念

mihomo
代理核心程序（原 Clash Meta）。mihomo-manager 管理的目标进程。
避免：Clash / 代理软件
mihomo-manager
本仓库的工具，提供 CLI + TUI 来管理 mihomo 实例的完整生命周期。
避免：管理器 / 前端
实例（Instance）
运行在宿主机上的单一 mihomo 进程。由 systemd（Linux）或 launchd（macOS）管理。
避免：服务节点
实例状态（InstanceState）
实例的生命周期状态：stopped → running → upgrading → failed。
避免：开启/关闭

安装生命周期

安装（Install）
下载 mihomo 二进制 → 部署到 /opt/mihomo/bin/ → 注册系统服务 → 写入默认配置 → 启动。
升级（Upgrade）
下载新版本 → 停止实例 → 备份旧二进制 → 替换 → 启动。启动失败时自动回滚。
卸载（Uninstall）
停止实例 → 注销系统服务 → 删除二进制/配置/数据。可指定 --keep-backup 保留配置备份。
回滚（Rollback）
升级过程中自动触发：新版本启动失败后，自动恢复旧二进制并重新启动。
避免：降级

配置管道

配置模板（Config Template）
/opt/mihomo/etc/config-template.yaml，含 {{subscription}} 和 {{routing_rules}} 占位符的模板文件。
订阅（Subscription）
远程代理节点列表，来自订阅 URL 或本地粘贴的配置内容。存储在 subscription-data.txt。
分流规则（Routing Rules）
用户自定义规则，决定哪些流量走哪个代理。存储在 routing-rules.txt，插入到 {{routing_rules}} 位置。
生成配置（Generated Config）
模板 + 订阅数据 + 分流规则合并后的最终 config.yaml，即 mihomo 实际加载的配置。

操作控制

启动 / 停止 / 重启 / 重载（Start / Stop / Restart / Reload）
重载仅热更新配置，不重启进程；重启完全停止再启动。
状态（Status）
返回实例是否已安装、当前实例状态、版本号。
版本列表（Versions）
从 GitHub Releases 获取最近 5 个可用版本，不含预发布版本（Prerelease 标签）。

文件系统布局

/opt/mihomo/bin/mihomo
mihomo 二进制文件。
/opt/mihomo/etc/config.yaml
mihomo 实际加载的生成配置。该文件会被定时覆盖，不要手动编辑。
/opt/mihomo/etc/config.yaml.bak.<timestamp>
订阅更新前自动生成的配置备份。
/opt/mihomo/etc/config-template.yaml
用户可编辑的配置模板（含 {{subscription}} / {{routing_rules}}）。
/opt/mihomo/etc/routing-rules.txt
用户自定义分流规则。
/opt/mihomo-manager/state/subscription-data.txt
订阅内容（URL 或原始数据），每次 subscription update 刷新。
/opt/mihomo-manager/state/subscription-url.txt
订阅 URL（可选），有 URL 时优先从远程拉取数据。
/opt/mihomo-manager/backups/mihomo.bak
升级时备份的旧二进制文件，用于回滚。
