mihomo-manager CLI & TUI 速查表

全局选项

Flag作用
--help显示帮助信息
--version显示版本号
--quiet抑制正常输出（仅输出错误）

CLI 命令

命令参数说明
install[version]安装 mihomo（默认 latest）
uninstall[--keep-backup]卸载，可选保留配置备份
start启动实例
stop停止实例
restart重启实例
reload热重载配置（不重启进程）
status查看状态和版本
upgrade[version]升级到指定版本（默认 latest）
versions列出最近 5 个可用版本
config preview预览生成的最终配置
template edit编辑配置模板（$EDITOR）
rules edit编辑分流规则（$EDITOR）
subscription set<data>设置订阅源（URL 或原始内容）
subscription update拉取远程订阅并更新配置
subscription schedule查看定时刷新状态
subscription schedule --interval<duration>设置定时刷新间隔（如 6h）
subscription schedule --off关闭定时刷新
logs[--tail=N] [--follow]查看 mihomo 日志

退出码

代码含义
0mihomo 正在运行
1已停止 / 升级中 / 操作失败
2未安装

TUI 键盘快捷键

快捷键作用可用条件
1Start 启动已安装且 Stopped
2Stop 停止已安装且 Running
3Restart 重启已安装且 Running

TUI 视图模式

模式说明
Status（仪表盘）默认视图，显示实例状态 + 四个操作按钮（Start/Stop/Restart/Reload）
Confirm Uninstall卸载确认对话框，通过 y/n 选择是否保留备份
Choose Version版本选择列表，用 ↑/↓ 导航，Enter 确认
Config四个子标签页：Subscription / Template / Rules / Preview，Tab 切换
Executing操作进行中的进度视图（安装/升级/卸载等），完成后自动返回 Status

日志查看选项

Flag作用示例
--tail=N显示最后 N 行（默认 50）mihomo-manager logs --tail=200
--follow持续跟踪新日志mihomo-manager logs --follow

典型工作流

# 1. 首次使用：安装
sudo mihomo-manager install

# 2. 设置订阅
mihomo-manager subscription set "https://example.com/sub"

# 3. 编辑模板（可选）
mihomo-manager template edit

# 4. 编辑分流规则（可选）
mihomo-manager rules edit

# 5. 更新配置
mihomo-manager subscription update

# 6. 设置定时刷新（可选）
mihomo-manager subscription schedule --interval 6h

# 7. 查看状态
mihomo-manager status

# 8. 进入 TUI 日常操作
mihomo-manager

常见问题

  • 重载 vs 重启：修改模板/规则后只需 reload 即可热更新，不需要 restart。
  • 二进制回滚：升级如果启动失败会自动回滚到旧版本，无需手动干预。
  • 配置备份：每次 subscription update 前自动备份 config.yaml.bak.<timestamp>。
  • sudo 要求：只有 install、uninstall、upgrade 需要 sudo，日常启停不需要。
