Lesson 2: 配置管理

完成本节后你将能管理 mihomo 的完整配置管道：设置订阅、编辑模板、编写分流规则、预览和生效。

配置管道概览

mihomo-manager 使用三条输入生成最终的 config.yaml：

订阅数据 ↓ 配置模板
(含 {{subscription}} / {{routing_rules}}) ↓ 合并 ↓ config.yaml

分流规则通过 {{routing_rules}} 占位符插入模板。

1. 设置订阅源

订阅源可以是远程 URL 或本地粘贴的配置数据：

mihomo-manager subscription set "https://sub.example.com/profile"

数据存储在 /opt/mihomo-manager/state/subscription-data.txt。如果是 URL，每次 update 时从远程拉取。

注意：数据是原始代理配置内容，不是 mihomo 的最终配置。模板引擎会把它插入到 {{subscription}} 位置。

2. 编辑配置模板

mihomo-manager template edit

这会用 $EDITOR（默认 vi）打开 /opt/mihomo/etc/config-template.yaml。一个典型的模板：

port: 7890
socks-port: 7891
allow-lan: false
mode: Rule
log-level: info
external-controller: 127.0.0.1:9090

proxies:
{{subscription}}

proxy-groups:
  - name: Proxy
    type: select
    proxies:
      - DIRECT

rules:
  - MATCH,DIRECT
{{routing_rules}}

{{subscription}} 会被订阅数据替换，{{routing_rules}} 会被分流规则替换。

3. 编辑分流规则

mihomo-manager rules edit

分流规则是追加在 rules: 段的额外规则行。一个例子：

  - DOMAIN-SUFFIX,google.com,Proxy
  - DOMAIN-KEYWORD,youtube,Proxy
  - GEOIP,CN,DIRECT
技巧：模板中的 rules: 段包含 MATCH,DIRECT 兜底规则。你的分流规则在它前面插入。

4. 预览生成配置

在实际生效前，预览最终结果：

mihomo-manager config preview

输出等同于即将写入 /opt/mihomo/etc/config.yaml 的内容。

5. 生效配置

mihomo-manager subscription update

该命令依次：

  • 读取订阅源（必要时从远程拉取）
  • 备份当前 config.yaml → config.yaml.bak.<timestamp>
  • 合并模板+订阅+规则 → 写入 config.yaml
  • 重载 mihomo 配置（reload，不重启进程）

6. 定时刷新

# 每 6 小时自动拉取订阅并重载
mihomo-manager subscription schedule --interval 6h

# 查看当前定时设置
mihomo-manager subscription schedule

# 关闭定时
mihomo-manager subscription schedule --off

支持的时间单位：30m、2h、1h30m 等。

练习：完成一个配置流程

  1. 运行 mihomo-manager subscription set "https://example.com/sub"
  2. 运行 mihomo-manager template edit，确保模板包含 {{subscription}} 和 {{routing_rules}}
  3. 运行 mihomo-manager config preview，观察输出
  4. 运行 mihomo-manager subscription update 使配置生效
  5. 运行 mihomo-manager subscription schedule --interval 6h
